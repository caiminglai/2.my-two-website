### 1.3 密钥安全

**关键发现**（`backend/db/index.js` 第420-422行）：

```javascript
const _tokenSeed = process.env.TOKEN_SECRET || process.env.ADMIN_PASSWORD || 'default';
const TOKEN_SECRET = process.env.TOKEN_SECRET || crypto.createHash('sha256').update('match-platform-secret-key-' + _tokenSeed).digest('hex');
const ENCRYPTION_KEY = process.env.ENCRYPTION_KEY || crypto.createHash('sha256').update('contact-encryption-key-' + _tokenSeed).digest();
```

**风险**：未设置环境变量时使用默认种子 `'default'`，攻击者可预测密钥。

**严重程度**：🔴 **高危**

#### 🔴 高危 - 支付宝签名验证使用私钥而非公钥

**文件**：`backend/routes/payment.js` 第131-149行

```javascript
function handleAlipayNotify(params) {
  const sign = params.sign;
  delete params.sign;
  delete params.sign_type;
  const calculatedSign = alipaySign(params, alipayConfig.privateKey);  // ❌ 使用私钥
  const verify = crypto.createVerify('RSA-SHA256');
  const stringToVerify = ...;
  verify.update(stringToVerify);
  if (!verify.verify(alipayConfig.alipayPublicKey, sign, 'base64'))  // ✅ 这行是对的
    return { success: false, message: '签名验证失败' };
}
```

**分析**：`calculatedSign` 使用私钥签名，但从未使用。实际验证使用 `alipayPublicKey` 是正确的。但代码逻辑混乱，存在误导性。

**结论**：虽然实际验证逻辑正确，但代码结构混乱，`calculatedSign` 变量完全无用，容易造成维护误解。

### 1.7 前端安全

#### 🔴 高危 - Auth Token存储在 localStorage

**文件**：`app/src/pages/LoginPage.tsx` 第88-91行

```javascript
localStorage.setItem('auth_token', data.token);
localStorage.setItem('user_id', data.user?.id || '');
localStorage.setItem('user_phone', data.user?.phone || '');
localStorage.setItem('user_nickname', data.user?.nickname || '');
```

**风险**：
- localStorage 可被 XSS 攻击读取
- Token可被窃取并用于冒充用户
- 用户手机号存储在 localStorage

**修复建议**：改用 HttpOnly Cookie 或 sessionStorage，配合 CSRF 防护。

---

#### 🟡 中危 - 批量同步用户数据无认证

**文件**：`app/src/services/user.service.ts` 第92-101行

```javascript
export function saveRows(rows: Row[]) {
  localStorage.setItem(LS_ROWS, JSON.stringify(rows));
  try {
    fetch(`${API_BASE_URL}/users/batch`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(rows)
    });
  } catch (e) { /* ignored */ }
}
```

**风险**：向 `/users/batch` 发送请求时未携带 Authorization Header，后端若接受无认证请求则可能被滥用。

### 1.8 高危问题详情

#### 🔴 高危 - Token和加密密钥默认可被预测

**文件**：`backend/db/index.js`

未设置环境变量时使用默认种子，攻击者可预测密钥并伪造Token。

**修复建议**：生产环境强制设置环境变量，移除默认值回退。

---

#### 🔴 高危 - check-unlock接口无需认证获取联系方式

**文件**：`backend/routes/contact-unlock.js`

`viewerId` 直接从 URL 参数获取，未验证用户身份，可遍历获取明文联系方式。

**修复建议**：从 `Authorization` header 获取 viewerId。

---

#### 🔴 高危 - Auth Token存储在 localStorage

**文件**：`app/src/pages/LoginPage.tsx`

localStorage 可被 XSS 攻击读取，导致 Token 泄露。

**修复建议**：改用 HttpOnly Cookie。

---

#### 🟡 中危 - 用户列表/搜索/创建接口公开访问

**文件**：`backend/server.js`

`GET /api/users`、`GET /api/users/search`、`POST /api/users` 公开访问，隐私信息可被爬取。

**修复建议**：增加认证要求或严格速率限制。

---

#### 🟡 中危 - 速率限制基于内存，多实例部署失效

**文件**：`backend/server.js`

所有速率限制基于内存 Map，多实例部署时完全失效。

**修复建议**：改用Redis。

---

#### 🟡 中危 - 用户数据一次可拉取1000条

**文件**：`backend/routes/users.js`

普通用户一次最多可获取 1000 条用户数据，易被批量爬取。

**修复建议**：普通用户限制为 50-100 条/页。

---

#### 🟡 中危 - Math.random()用于Token生成

**文件**：`backend/db/index.js`

Token生成使用 `Math.random()`，不是加密安全的随机数生成器。

**修复建议**：改用 `crypto.randomBytes`。
