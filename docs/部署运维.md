## 5. Nginx 配置

配置文件：`/etc/nginx/conf.d/my-website.conf`

### 5.1 your_other_domain.com 完整配置（v6.2 生产环境）

```nginx
# ====== 域名：your_other_domain.com（精准匹配，主域名部署）======
server {
    listen 443 ssl;
    server_name your_other_domain.com www.your_other_domain.com;

    # SSL 证书（Let's Encrypt，自动续期）
    ssl_certificate /etc/letsencrypt/live/your_other_domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your_other_domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 精准匹配 - 前端静态文件（根路径）
    location / {
        root /var/www/my-two-website/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 精准匹配 - API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 精准匹配 - 管理后台
    location /admin {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 5.2 HTTP 自动跳转 HTTPS

```nginx
# ====== HTTP 自动跳转 HTTPS ======
server {
    listen 80;
    server_name your_domain.com www.your_domain.com your_other_domain.com www.your_other_domain.com;
    return 301 https://$host$request_uri;
}
```

### 5.3 子路径部署配置（旧版 /jzxr/ 兼容）

如需保留子路径访问方式（如通过 your_domain.com/jzxr/ 访问），在对应 server 块中添加：

```nginx
# 补斜杠（重要！访问 /jzxr 自动跳转 /jzxr/）
location = /jzxr { return 301 /jzxr/; }

# 前端 SPA
location /jzxr/ {
    alias /var/www/my-two-website/dist/;
    index index.html;
    try_files $uri $uri/ @jzxr_fallback;
}
location @jzxr_fallback {
    root /var/www/my-two-website/dist;
    try_files /index.html =404;
}

# API 代理（带子路径 rewrite）
location /jzxr/api/ {
    rewrite ^/jzxr/api/(.*)$ /api/$1 break;
    proxy_pass http://127.0.0.1:8080;
}

# 管理后台（HTML 页面）
location /jzxr/admin {
    rewrite ^/jzxr/admin(.*)$ /admin$1 break;
    proxy_pass http://127.0.0.1:8080;
}
location /jzxr/admin/ {
    rewrite ^/jzxr/admin/(.*)$ /admin/$1 break;
    proxy_pass http://127.0.0.1:8080;
}

# 用户上传文件
location /jzxr/uploads/ { alias /var/www/my-two-website/backend/uploads/; }

# 收款码图片
location /wechat-qr.png { alias /var/www/my-two-website/dist/wechat-qr.png; }
location /alipay-qr.png { alias /var/www/my-two-website/dist/alipay-qr.png; }
```

### 5.4 Nginx 常用命令

## 9. 部署流程（每次改代码后）

第一步：本地修改代码，确保所有功能在本地开发环境测试正常。

第二步：构建前端
```bash
cd app
npm run build
```

第三步：上传文件到服务器（使用 WinSCP）
- `app/dist/` 内所有文件 -> `/var/www/my-two-website/dist/`
- 如果改了后端代码：`backend/` 对应文件 -> `/var/www/my-two-website/backend/`

第四步：服务器端操作
```bash
cd /var/www/my-two-website/backend
# 如果有新依赖
rm -rf node_modules && npm install
# 重启后端
pm2 restart my-two-website --update-env
# 如果改了 Nginx 配置
sudo nginx -t
sudo systemctl reload nginx
```

第五步：验证部署
浏览器访问 `https://your_other_domain.com/`，按 Ctrl+F5 强制刷新。

## 10. 检查清单

### 10.1 前端自查

| 检查项 | 文件 | 检查内容 |
|:---|:---|:---|
| Vite 代理路径 | `vite.config.ts` | `/jzxr/api` 代理是否正确 rewrite |
| Router basename | `App.tsx` | 必须与部署路径一致 |
| Vite base | `vite.config.ts` | 必须与部署路径一致 |
| API 请求路径 | `src/api/*.ts` | 必须使用正确的 API 前缀 |
| 环境变量 | `.env.development` | `VITE_API_BASE_URL` 正确 |
| 不写死 localhost | 所有代码 | 禁止 `localhost:8080`、`127.0.0.1` |

### 10.2 后端自查

| 检查项 | 文件 | 检查内容 |
|:---|:---|:---|
| `/admin/main` 路由 | `server.js` | 确保不被静态文件服务拦截 |
| 登录 API 路径 | `admin/login.html` | 路径正确 |
| 登录跳转路径 | `admin/login.html` | 跳转到管理主页 |
| 数据 API 路径 | `admin/main.html` | API 路径前缀正确 |

### 10.3 部署后自查

1. 访问 `https://your_other_domain.com/` 测试页面加载
2. 打开浏览器 DevTools -> Network，检查 JS 文件是否按需加载
3. 滚动用户列表，观察图片是否懒加载
4. 触发错误操作，验证错误提示是否友好
5. 查看数据库 users 表，确认 contact 字段已加密

### 10.4 常见错误根因速查

| 现象 | 根因 | 解决 |
|:---|:---|:---|
| 前端 404 | Vite 代理配置错误 | 检查 rewrite 规则 |
| 前端数据空 | API 路径错误 | 检查路径前缀 |
| `/jzxr/admin` 404 | server.js 路由顺序 | `/admin` 路由放静态文件服务前面 |
| `/jzxr/admin/main` 404 | 静态文件服务拦截 | 排除 `/admin/main` |
| 管理页面数据空 | API 路径前缀错误 | 检查 API 路径 |
| 502 Bad Gateway | 后端没启动 | `pm2 start my-two-website` |
| 端口占用 | - | `taskkill /f /im node.exe`（Windows） |
| 资源 404 | base 路径斜杠问题 | 检查 `VITE_BASE_URL` |
| 无用户数据 | - | 重启后端 + 刷新页面 |
| 上传失败 | - | 检查 `backend/uploads/` 目录是否存在 |
| Token 无效 | - | 检查 `backend/.env` 中的 Token 配置 |
| `.env` 修改不生效 | 没加 `--update-env` | `pm2 restart my-two-website --update-env` |

### 10.5 关键路径前缀

所有路径必须与部署方式匹配：your_other_domain.com 根路径部署用 `/`，子路径部署用 `/jzxr`。

## 11. 故障排查速查

### 11.1 快速诊断流程

```
现象：网站打不开（502）
  |
检查后端是否启动：pm2 list
  |
查看后端日志：pm2 logs my-two-website
  |
检查 Nginx 配置：nginx -t
  |
检查 Nginx 错误日志：tail -f /var/log/nginx/error.log
```

### 11.2 常见问题速查表

| 现象 | 原因 | 解决方法 |
|:---|:---|:---|
| 网站打不开（502） | 后端服务没启动 | `pm2 start my-two-website` |
| API 返回 404 | Nginx 代理配置错误 | 检查 `/etc/nginx/conf.d/my-website.conf` |
| API 返回 500 | 数据库表不存在或缺字段 | 重启后端让 `initTables()` 自动建表 |
| `.env` 修改不生效 | 没重启或没加 `--update-env` | `pm2 restart my-two-website --update-env` |
| `/jzxr` 不加斜杠进不去 | Nginx 没配重定向 | 检查 `location = /jzxr { return 301 /jzxr/; }` |
| 前端刷新后 404 | SPA fallback 没配置 | 检查 Nginx 的 fallback 配置 |
| 管理后台登录失败 | API 路径错误或 Token 无效 | 检查 Nginx 的 admin 配置 |
| 图片上传失败 | uploads 目录权限问题 | `chmod 755 /var/www/my-two-website/backend/uploads` |
| 管理后台按钮无反应 | main.html JS 语法错误 | F12 控制台定位报错行号 |
| 前端数据空但后端正常 | API 路径前缀错误 | 检查前端请求路径 |
| 上传失败 500 | DEPOSIT_RULES 未定义 | 检查 `routes/upload.js` 顶部导入 |
| 举报列表显示"未知" | 返回字段名不匹配 | 检查 `routes/reports.js` 字段映射 |
| 向量搜索报错 | PostgreSQL/pgvector 未安装 | 确认 PostgreSQL 18 + pgvector v0.8.2 已安装 |
| 向量生成失败 | EMBEDDING_API_KEY 未配置 | 检查 `.env` 中 EMBEDDING_PROVIDER 和 API_KEY |

### 11.3 本地开发问题

| 症状 | 根因 | 解决 |
|------|------|------|
| npm run dev 报错 ENOENT | 没在 app/ 文件夹执行 | `cd E:\website\2.my-two-website\app` |
| npm start 报错 ENOENT | 没在 backend/ 文件夹执行 | `cd E:\website\2.my-two-website\backend` |
| 后端 500 / 请求超时 | 后端卡住或数据库异常 | Ctrl+C -> 重新 npm start |
| 页面能开但无数据 | API 超时 | Ctrl+F5 + 重启后端 |
| better-sqlite3 编译失败 | 缺 C++ 编译器 | 装 VS Build Tools 或确认 Node v20 |
| 端口占用 EADDRINUSE | 旧进程没释放 | `taskkill /f /im node.exe` |
| PostgreSQL 连接失败 | 服务未启动或密码错误 | 检查 PostgreSQL 服务状态和 `.env` 中 PG_PASSWORD |
| pgvector 扩展未找到 | 未安装 pgvector 扩展 | 在数据库中执行 `CREATE EXTENSION vector;` |

### 11.4 调试铁律

1. 本地启动必须先 cd 进对应文件夹（app/ 或 backend/）
2. 改 .env 后必须 pm2 restart --update-env
3. Node 版本必须 v20.20.2，不能回退到 v22
4. 生产构建前检查 VITE_BASE_URL 与实际部署方式一致
5. Windows 本地重装 node_modules 后如果 better-sqlite3 报错，确认已装 VS C++ Build Tools
6. 服务器只执行三件事：接收代码、安装依赖、重启服务。不在服务器上临时改代码
7. 修改后端路由代码后必须重启服务
8. 所有服务器操作前先备份配置文件：`cp /etc/nginx/conf.d/xxx.conf /etc/nginx/conf.d/xxx.conf.bak`
