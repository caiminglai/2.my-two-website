# 第一部分：项目概述

## 1.1 项目定位

精准寻人是一个相亲匹配平台，核心功能：

- 用户资料存储与展示
- 用户关系/匹配算法
- 快速筛选搜索
- 温馨暖色调 UI
- 支付集成（微信/支付宝）

## 1.2 域名与品牌

| 域名 | 用途 |
|------|------|
| your_domain.com | 精准寻人主域名 |
| your_other_domain.com/jzxr/ | 同物异名域名下的相亲平台入口 |

# 第二部分：技术栈

## 2.1 推荐技术组合

```
用户资料存储  → PostgreSQL / MySQL（结构化数据）
用户关系/匹配 → Neo4j（谁和谁是潜在匹配）
快速筛选搜索  → Meilisearch（多条件匹配）
前端展示      → 纯 HTML + JS（温馨暖色调 UI）
后端 API      → Node.js 原生 http 模块
部署          → 阿里云服务器 + Nginx
```

## 2.2 当前技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React + Vite |
| 后端 | 原生 Node.js http 模块 + SQLite (better-sqlite3) |
| 数据库 | match.db |
| 部署 | 阿里云 + Nginx + PM2 |

## 4.6 Nginx 配置（your_domain.com 部分）

```nginx
server {
    listen 80;
    server_name your_domain.com www.your_domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your_domain.com www.your_domain.com;

    ssl_certificate /etc/letsencrypt/live/your_domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your_domain.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    location = / {
        return 302 /jzxr/;
    }

    location = /jzxr {
        return 301 /jzxr/;
    }

    location /jzxr/ {
        alias /var/www/my-two-website/dist/;
        index index.html;
        try_files $uri $uri/ @jzxr_fallback;
    }

    location @jzxr_fallback {
        root /var/www/my-two-website/dist;
        try_files /index.html =404;
    }

    location /jzxr/api/ {
        rewrite ^/jzxr/api/(.*)$ /api/$1 break;
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /jzxr/admin {
        rewrite ^/jzxr/admin(.*)$ /admin$1 break;
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

# 第五部分：问题排查

## 5.1 your_domain.com JS 加载问题

### 问题现象
页面空白，控制台报错：`Failed to load module script: Expected a JavaScript-or-Wasm module script but the server responded with a MIME type of "text/html".`

### 修复方法

**方法 A（推荐）：修改站点配置**
在 `/etc/nginx/conf.d/` 对应配置文件的 `server { }` 块内添加：
```nginx
location ~* \.js$ {
    add_header Content-Type application/javascript;
}
```

**方法 B：检查 mime.types**
确认 `/etc/nginx/mime.types` 中存在：
```
application/javascript                js;
```

修复后执行 `sudo nginx -t && sudo systemctl restart nginx`。

### 验证
清除浏览器缓存（Ctrl+Shift+R），重新访问，F12 Network 标签确认 .js 文件 Content-Type 为 `application/javascript`。

## 5.2 本地开发问题

| 症状 | 根因 | 解决 |
|------|------|------|
| npm run dev 报错 ENOENT | 没在 app/ 文件夹执行 | `cd E:\website\2.my-two-website\app` |
| npm start 报错 ENOENT | 没在 backend/ 文件夹执行 | `cd E:\website\2.my-two-website\backend` |
| 后端 500 / 请求超时 | 后端卡住或数据库异常 | Ctrl+C → 重新 npm start |
| 页面能开但无数据 | API超时 | Ctrl+F5 + 重启后端 |
| better-sqlite3 编译失败 | 缺C++编译器或Node版本不对 | 装 VS Build Tools 或确认 Node v20 |
| 端口占用 EADDRINUSE | 旧进程没释放 | `taskkill /f /im node.exe` |
| jzxr 资源404 | base路径斜杠问题 | 检查 .env.production 的 VITE_BASE_URL |

## 5.3 服务器问题

| 症状 | 根因 | 解决 |
|------|------|------|
| 网页打不开 | Nginx没启动 | `systemctl start nginx && nginx -t` |
| 页面能开但没数据 | 后端挂了 | `pm2 list` → `pm2 restart my-two-website` |
| 登录失败 | .env改了没重启 | `pm2 restart my-two-website --update-env` |
| 页面图片/样式丢失 | 路径没加斜杠 | 检查网址 `/jzxr/` |
| 改完代码没变化 | 没重启或缓存 | Ctrl+F5 + pm2 restart |
| 502 Bad Gateway | 后端端口不匹配 | `pm2 list` + `ss -tlnp | grep 8080` |

## 5.4 调试铁律

1. 本地启动必须先 cd 进对应文件夹（app/ 或 backend/）
2. 改 .env 后必须 pm2 restart --update-env
3. Node 版本必须 v20.20.2，不能回退到 v22
4. jzxr 生产构建前检查 VITE_BASE_URL 带斜杠
5. Windows 本地重装 node_modules 后如果 better-sqlite3 报错，确认已装 VS C++ Build Tools
6. 服务器只执行三件事：接收代码、安装依赖、重启服务。不在服务器上临时改代码
7. 修改后端路由代码后必须重启服务（先 netstat 找端口 PID → taskkill 杀旧进程 → 启动新进程）
8. 所有服务器操作前先备份配置文件：`cp /etc/nginx/conf.d/xxx.conf /etc/nginx/conf.d/xxx.conf.bak`
